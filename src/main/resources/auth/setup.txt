Auto import realm (docker):
docker run --name evatool-test-auth -p 8081:8080 -v ./keycloak:/opt/jboss/keycloak/imports -e KEYCLOAK_USER=admin -e KEYCLOAK_PASSWORD=admin -e KEYCLOAK_IMPORT=/opt/jboss/keycloak/imports/evatool-realm.json quay.io/keycloak/keycloak:13.0.0




docker run -e KEYCLOAK_USER=<USERNAME> -e KEYCLOAK_PASSWORD=<PASSWORD> \
    -e KEYCLOAK_IMPORT=/tmp/example-realm.json -v /tmp/example-realm.json:/tmp/example-realm.json jboss/keycloak






Start keycloak with http (docker):
docker run --name evatool-test-auth -p 8081:8080 -e KEYCLOAK_USER=admin -e KEYCLOAK_PASSWORD=admin quay.io/keycloak/keycloak:13.0.0

Start keycloak with https (docker):
docker run --name evatool-test-auth -p 8081:8443 -e KEYCLOAK_USER=admin -e KEYCLOAK_PASSWORD=admin quay.io/keycloak/keycloak:13.0.0



Start keycloak (without docker):
standalone.bat -Djboss.socket.binding.port-offset=1

Import realm:
Click on add realm and then select the file as import



Start keycloak to export realm with users:
standalone.bat -Djboss.socket.binding.port-offset=1 -Dkeycloak.migration.action=export -Dkeycloak.migration.provider=singleFile -Dkeycloak.migration.file=evatool-realm.json -Dkeycloak.migration.realmName=evatool-realm -Dkeycloak.migration.usersExportStrategy=REALM_FILE



Master Realm:
Enable Web Origins (Cors) Clients -> admin-cli -> Web Origins: * // TODO Automate this

Then go to:
http://localhost:8081/auth/

https://localhost:8081/auth/
